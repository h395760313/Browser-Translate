# Chrome 翻译插件 UI 设计方案

## 1. 设计理念与风格定位

### 1.1 产品定位
Chrome 翻译插件是一款嵌入浏览器的轻量级翻译工具，目标用户为需要频繁跨语言阅读的普通用户。核心场景是用户选中网页文字后，通过弹出框即时获取翻译结果。

### 1.2 设计理念
**"无感融入，轻盈高效"** —— 插件不应抢夺用户对网页内容的注意力，翻译框应是网页的有机延伸，而非独立存在的界面。
- **极简克制**：只在必要时出现，翻译完成后不造成视觉干扰
- **即时响应**：选词即翻译，无需额外点击（自动模式）
- **功能纯粹**：翻译 + 朗读，两大核心功能做到极致

### 1.3 风格定位
**AI-Native 风格 + 工具类产品的务实美学**：
- 对话式 / 智能助手的气质 — 温和、专业、可信赖
- 大量留白，内容优先，无导航杂乱
- 深浅模式自适应（跟随 Chrome 主题）

### 1.4 参考基准
参考 Google Translate 插件、DeepL 插件、OpenL 的设计语言，融合浏览器扩展的交互规范。

## 2. 整体配色方案

### 2.1 颜色语义化 Token

| Token | 用途 | 浅色模式 | 深色模式 |
|-------|------|---------|---------|
| `--primary` | 品牌色 / 主要操作 | `#2563EB` | `#3B82F6` |
| `--secondary` | 次要元素 | `#3B82F6` | `#60A5FA` |
| `--accent` | 强调 / CTA 按钮 | `#F97316` | `#FB923C` |
| `--bg` | 背景色 | `#FFFFFF` | `#1E293B` |
| `--surface` | 卡片 / 弹出框背景 | `#FFFFFF` | `#334155` |
| `--surface-elevated` | 悬浮层背景 | `#FFFFFF` | `#475569` |
| `--text-primary` | 主要文字 | `#1E293B` | `#F8FAFC` |
| `--text-secondary` | 次要文字 | `#64748B` | `#94A3B8` |
| `--text-muted` | 占位文字 | `#94A3B8` | `#64748B` |
| `--border` | 边框 / 分隔线 | `#E2E8F0` | `#475569` |
| `--shadow-color` | 阴影颜色 | `rgba(0,0,0,0.08)` | `rgba(0,0,0,0.3)` |
| `--success` | 成功状态 | `#10B981` | `#34D399` |
| `--error` | 错误状态 | `#EF4444` | `#F87171` |

### 2.2 色彩使用规范
- **翻译框主体**：使用 `--surface`，与网页背景保持适度对比
- **朗读按钮**：使用 `--primary` 色系，强调为辅助功能
- **复制 / 收藏按钮**：使用 `--text-secondary`，低视觉权重
- **CTA 操作**（如"开启自动翻译"引导）：使用 `--accent`，高对比度
- **原文与译文的视觉区分**：原文使用 `--text-secondary`，译文使用 `--text-primary`

## 3. 字体系统

### 3.1 字体选择
```
@import url('https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;500;600;700&display=swap');

font-family: 'Plus Jakarta Sans', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

### 3.2 字号层级

| 层级 | 字号 | 字重 | 用途 | 行高 |
|------|------|------|------|------|
| 标题 (title) | 16px | 600 | 弹出框标题（语言标签） | 1.4 |
| 正文 (body) | 14px | 400 | 翻译结果展示 | 1.6 |
| 辅助 (caption) | 12px | 400 | 提示文字、音标、字符数 | 1.5 |
| 按钮 (button) | 13px | 500 | 操作按钮文字 | 1.0 |

### 3.3 字号规范
- **正文翻译结果**：14px，不得小于 12px
- **自动翻译模式下翻译内容**：允许 13px（内容密集场景）
- **长段落自动换行**：不截断，支持 3 行最大高度

## 4. 界面布局与组件设计

### 4.1 翻译弹出框（Translation Popover）

#### 4.1.1 尺寸规范

| 属性 | 移动端 (<768px) | 桌面端 (≥768px) |
|------|----------------|----------------|
| 最大宽度 | `calc(100vw - 32px)` | 420px |
| 最小宽度 | 280px | 320px |
| 圆角 | 16px | 16px |
| 内边距 | 16px | 20px |
| 阴影 | `0 8px 32px rgba(0,0,0,0.12)` | `0 12px 48px rgba(0,0,0,0.15)` |

#### 4.1.2 弹出位置逻辑
1. **首选位置**：选中文字的正下方（向下箭头指向触发词）
2. **备用位置**（空间不足时）：选中文字的正上方
3. **水平适配**：若右侧空间不足，向左偏移；左侧同理
4. **距离视口边缘**：最小边距 8px，防止贴边

#### 4.1.3 弹出框结构

```
┌──────────────────────────────┐
│ [语言方向标签]        [×]   │  ← Header: 16px 高度
├──────────────────────────────┤
│                              │
│  翻译结果文本（大字号，易读） │  ← Translation Content
│                              │
│  [音标 / 拼音 / 词性]        │  ← Meta info (可选)
│                              │
├──────────────────────────────┤
│ [🔊朗读] [📋复制] [⭐收藏]   │  ← Action Bar
└──────────────────────────────┘
  ↑ 底部小三角指向触发词
```

#### 4.1.4 组件状态

| 状态 | 视觉表现 |
|------|---------|
| **加载中** | 内容区显示 3 个骨架行（pulse 动画），操作按钮禁用态 |
| **翻译成功** | 翻译结果淡入显示（fade-in 200ms），所有按钮可交互 |
| **翻译失败** | 显示错误图标 + 简短文案（"翻译失败，请重试"）+ 重试按钮 |
| **无结果** | 显示"未找到翻译结果"空状态 |
| **朗读中** | 朗读按钮变为播放中状态（图标变为暂停图标，按钮高亮） |

### 4.2 翻译结果展示区域

#### 4.2.1 布局结构
- **语言方向标签**：左上角，如"EN → 中文"，字号 12px，颜色 `--text-muted`，字重 500
- **翻译文本**：字号 14px，行高 1.6，颜色 `--text-primary`
- **元信息区**（可选，根据翻译类型显示）：
  - 音标：`(/)` 格式，12px，`--text-secondary`
  - 词性：斜体 `noun / 动词`，12px，`--primary` 色
  - 词数统计：底部右侧，11px，`--text-muted`

#### 4.2.2 多义词展示
当存在多个翻译结果时，采用折叠列表：

```
翻译结果文本
  释义1 · 词性   ▼  (可展开)
  释义2 · 词性
  释义3 · 词性
```

- 默认展开第一项
- 折叠时最多显示 1 行摘要
- 展开动画：高度过渡 200ms ease-out

#### 4.2.3 长文本处理
- 最大显示 3 行（桌面端），超出部分显示"展开更多"
- "展开更多" 链接在底部，12px，underline on hover
- 展开后支持滚动，滚动区域最大高度 200px（带原生滚动条样式）

### 4.3 朗读按钮设计

#### 4.3.1 视觉规格
- **图标**：SVG 扬声器图标（Lucide `volume-2`），24×24px
- **按钮尺寸**：36×36px 触控区域，图标 20×20px 居中
- **背景**：默认透明，hover 时 `--primary` 背景色淡至 10% 透明度
- **圆角**：8px（与整体弹出框协调）
- **颜色**：`--primary`

#### 4.3.2 按钮状态

| 状态 | 视觉表现 |
|------|---------|
| **默认** | 透明背景，primary 色图标 |
| **Hover** | primary 色背景（10% 透明度），图标颜色加深 |
| **按下** | primary 色背景（15% 透明度），scale(0.95) |
| **播放中** | 图标变为 `volume-x`（静音图标），背景 primary 色（15%） |
| **加载中** | 图标旋转 360°（loading 动画），不可点击 |
| **禁用** | 透明度 40%，cursor: not-allowed |

#### 4.3.3 交互行为
- **点击行为**：点击后立即响应，开始朗读对应文本（原文或译文可切换）
- **朗读范围**：支持切换"朗读原文" / "朗读译文"
- **多段文本**：连续朗读，朗读完成后自动恢复为默认图标
- **朗读控制**：朗读中途再次点击 = 停止朗读

### 4.4 操作按钮栏（Action Bar）

#### 4.4.1 按钮组布局
三个操作按钮水平排列，等宽分布，使用 4px 间隔：
```
[🔊 朗读] [📋 复制] [⭐ 收藏]
```

#### 4.4.2 按钮设计规范
| 按钮 | 图标 | 主色 | 副色 |
|------|------|------|------|
| 朗读 | `volume-2` / `volume-x` | `--primary` | `#2563EB` |
| 复制 | `copy` / `check` | `--text-secondary` | `#10B981`（成功后） |
| 收藏 | `star` / `star-filled` | `--text-secondary` | `#F59E0B`（收藏后） |

#### 4.4.3 复制按钮特殊行为
- 点击后：图标变为 `check`，按钮短暂变绿（200ms）
- 同时显示 Tooltip："已复制"（1.5s 后自动消失）
- 1.5s 后恢复默认状态

## 5. 交互流程与动画

### 5.1 核心交互流程

```
用户选中文字
    ↓
检测翻译触发（自动模式 → 立即请求 | 手动模式 → 显示"译"按钮）
    ↓
显示加载状态（骨架屏，150ms 内出现）
    ↓
接收翻译结果 → 淡入动画显示（200ms ease-out）
    ↓
用户可交互（朗读/复制/收藏）
    ↓
点击空白处 或 ESC → 弹出框消失（150ms fade-out）
```

### 5.2 弹出框动画

#### 5.2.1 出现动画（Enter）
- **时长**：200ms
- **缓动函数**：`cubic-bezier(0.16, 1, 0.3, 1)` (ease-out)
- **效果**：从触发源（选中文字区域）向目标位置缩放 + 淡入
- **空间连续性**：底部小三角始终指向选中文字的中心
- **CSS 实现**：
  ```css
  transform-origin: center top;
  animation: popover-enter 200ms cubic-bezier(0.16, 1, 0.3, 1) forwards;
  @keyframes popover-enter {
    from { opacity: 0; transform: scale(0.92) translateY(-8px); }
    to   { opacity: 1; transform: scale(1)    translateY(0); }
  }
  ```

#### 5.2.2 消失动画（Exit）
- **时长**：150ms（比出现快，约 75%）
- **缓动函数**：`cubic-bezier(0.4, 0, 1, 1)` (ease-in)
- **效果**：淡出 + 轻微上移
- **CSS 实现**：
  ```css
  animation: popover-exit 150ms cubic-bezier(0.4, 0, 1, 1) forwards;
  @keyframes popover-exit {
    from { opacity: 1; transform: scale(1)    translateY(0); }
    to   { opacity: 0; transform: scale(0.95) translateY(-4px); }
  }
  ```

#### 5.2.3 位移动画
当弹出框从下方切换到上方时：
- **时长**：250ms
- **缓动函数**：`ease-in-out`
- **效果**：垂直方向平滑滑动，带轻微弹性

### 5.3 翻译结果加载动画

#### 5.3.1 骨架屏（Skeleton）
- **时机**：请求发出后 100ms 仍无响应时显示
- **形状**：3 条矩形骨架行
  - 第一行：宽度 90%，高度 14px，圆角 6px
  - 第二行：宽度 75%，高度 14px，圆角 6px
  - 第三行：宽度 60%，高度 14px，圆角 6px
- **动画**：shimmer 效果，从左到右渐变扫描
- **CSS**：
  ```css
  background: linear-gradient(90deg,
    var(--skeleton-base) 25%,
    var(--skeleton-shine) 50%,
    var(--skeleton-base) 75%
  );
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  ```

#### 5.3.2 流式文字效果（可选）
若后端支持流式返回：
- 翻译结果逐字符/逐词出现，打字机效果
- 每字符间隔 20-30ms（可感知但不慢）
- 光标显示在文字末尾（小竖线，1s 闪烁动画）
- 光标在接收完毕后 300ms 消失

### 5.4 朗读动画

#### 5.4.1 播放中状态
- **图标动画**：音量图标内的音波从 2 条变为 3 条，循环播放（800ms ease-in-out）
- **按钮背景**：微微发光效果（box-shadow 扩散动画）

#### 5.4.2 停止 / 完成动画
- 图标平滑从 `volume-2` 过渡到默认状态（150ms）
- 背景发光淡出（200ms）

### 5.5 微交互动画

| 交互 | 效果 | 时长 | 缓动 |
|------|------|------|------|
| 按钮 Hover | 背景色淡入，图标微微放大 | 150ms | ease-out |
| 按钮按下 | scale(0.95) | 100ms | ease-out |
| 按钮释放 | scale(1) 回弹 | 150ms | cubic-bezier(0.34, 1.56, 0.64, 1) |
| 收藏成功 | 图标填充 + 轻微弹跳 | 300ms | spring |
| 复制成功 | 图标切换 + 绿色 flash | 200ms | ease-out |
| 错误提示 | 从下方滑入 | 200ms | ease-out |
| Toast 消失 | 向上淡出 | 200ms | ease-in |

### 5.6 无障碍动画设置

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 6. 弹出框各状态设计

### 6.1 状态总览

| 状态 | 触发条件 | 视觉表现 |
|------|---------|---------|
| **就绪** | 插件已启用，等待用户操作 | 无弹出框 |
| **加载中** | 翻译请求发出后 | 骨架屏 + 按钮禁用 |
| **翻译成功** | 收到翻译结果 | 结果内容 + 操作按钮可交互 |
| **翻译失败** | 网络错误 / API 错误 | 错误图标 + 错误文案 + 重试按钮 |
| **无结果** | 翻译服务无匹配结果 | 空状态提示 |
| **朗读中** | 正在朗读文本 | 朗读按钮播放状态 |
| **收藏成功** | 点击收藏按钮后 | 按钮变为填充星形 + Toast |
| **已复制** | 点击复制按钮后 | check 图标 + "已复制" Toast |

### 6.2 错误状态详细设计
- **图标**：`alert-circle`（Lucide），24px，`--error` 色
- **标题**：13px，`--text-primary`，如"翻译失败"
- **描述**：12px，`--text-secondary`，如"网络连接异常，请检查网络后重试"
- **重试按钮**：`--primary` 色，中等尺寸，居中
- **布局**：垂直居中，上下间距 12px

### 6.3 空状态详细设计
- **图标**：`search-x` 或 `text-search`（Lucide），24px，`--text-muted` 色
- **文案**：13px，`--text-secondary`，如"未找到相关翻译"
- **建议**（可选）：12px，提示用户"可尝试选择更完整的句子"

## 9. 技术实现规范

### 9.1 图标库
统一使用 **Lucide Icons**（stroke-width: 1.75）：
- 朗读图标：`volume-2`, `volume-x`
- 复制图标：`copy`, `check`
- 收藏图标：`star`, `star`
- 关闭图标：`x`
- 错误图标：`alert-circle`
- 设置图标：`settings`
- 语言图标：`languages`

### 9.2 CSS 变量体系
```css
:root {
  /* 尺寸 */
  --popover-radius: 16px;
  --popover-padding: 20px;
  --popover-max-width: 420px;
  --btn-size: 36px;
  --icon-size: 20px;

  /* 间距 */
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 24px;

  /* 动画 */
  --duration-fast: 100ms;
  --duration-normal: 200ms;
  --duration-slow: 300ms;
  --ease-out: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-in: cubic-bezier(0.4, 0, 1, 1);
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);

  /* 阴影 */
  --shadow-popover: 0 12px 48px rgba(0, 0, 0, 0.15);
  --shadow-button-hover: 0 2px 8px rgba(37, 99, 235, 0.2);

  /* 圆角 */
  --radius-sm: 6px;
  --radius-md: 8px;
  --radius-lg: 16px;
}
```

### 9.3 深色模式实现
- 使用 CSS 媒体查询 + `prefers-color-scheme`
- Chrome 扩展支持 `chrome.storage` 存储用户主题偏好
- 所有颜色通过 CSS 变量引用，不使用硬编码 hex

### 9.4 性能要点
- 弹出框使用 `transform` + `opacity` 实现动画（GPU 加速）
- 图标使用 SVG 内联，不依赖外部资源
- 字体使用 `font-display: swap` 避免 FOIT
- 骨架屏动画使用 `will-change: background-position`
